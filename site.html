Controle de Versão:

O Controle de versão é um sistem que localiza e gerencia mudanças realizadas em arquivos e em pastas ao longo do tempo.

Como Funciona:

Controle de Versão: Realiza a preferência e o gerenciamento nas versões dos arquivos dos projetos.

Registro de Mudança: Cada alteração é registrada como um commit e descreve uma mensagem mostrando as alterações feitas.

Colaboração: Concede permição que outras pessoas realizem o trabalho dentro do mesmo projeto simultaneamente.

Reversão: Realiza o facilitamento de versões antecedentes do projeto atual.

Controle de Mudança:

O Controle de mudanças é um processo onde envolvem identificação, documentação e o gerenciamento de alterações de um projeto. Ele demonstra apenas o código, mas também os requisitos e o funcionamento.

Identificação de mudanças: Estima e documenta as mudanças requeridas e propostas.

Avaliação: Avalia o quão crítico a mudança pode realizar no projeto.

Aprovação: Deixa que mudanças possam ser revisadas e aprovadas antes de serem inseridas ao código.

Implementação e Verificação: As mudanças aprovadas serão inseridas e verificadas para realizar a garantia de que foram implementadas conforme o esperado.

O objetivo é garantir de que todas as mudanças feitas ao código sejam realizadas de forma controlada e uniforme, para que o impacto realizado com a implementação seja mínima. Pois, a qualidade é ecessencial para o comprometimento com o orçamento do cliente.

Integração Contínua:

A Integração Contínua é a realização de uma prática de desenvolvimento de software onde o código é constantemente integrado para um repositório compartilhado, com uma realização de testes automatizados para detectar erros.

Commit Frequente: Desenvolvedores enviam mudanças para o repostório frequentemente

Build Automatizado: A cada alteração envia um processo de build Automatizada realizando uma compilação e a verificação do código.

Teste Automatizado: Realiza a inicialização dos testes automatizados para garantir que o código não quebre código já anteriormente integrados.

Feedback Rápido: Realiza o envio dos feedback instantantâneamente sobre a qualidade e integridade do código, permitindo com que problemas sejam indentificados.
